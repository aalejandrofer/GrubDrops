Authorize {{.DisplayName}}

Kick has no public OAuth. GrubDrops replays your kick.com session directly over a Chrome-fingerprinted HTTP client (utls) — no browser, no sidecar. When discovery logs cloudflare / 401, the cookies have gone stale — re-grab here.

recommended

Helper CLI / pre-built app

The helper reads your local browser's kick.com cookies (kick_session, XSRF-TOKEN, cf_clearance, session_token) and pushes them. No DevTools, no copy-paste.

one-liner
go run ./cmd/dropsminer-helper kick {{.AccountID}} --channel STREAMER[,STREAMER2,...]
Pass one or more Kick channel logins (comma-separated, or repeat --channel). Add --miner URL for a remote deployment, --password PW to skip the prompt.

No Go installed? Download the pre-built helper:

⬇ macOS (Apple Silicon) ⬇ macOS (Intel) ⬇ Windows ⬇ Linux

Then run it from a terminal: grubdrops-helper-… kick {{.AccountID}} --channel STREAMER --miner <your GrubDrops URL> (macOS/Linux: chmod +x the file first).

fallback

Manual cookie paste

  1. Open kick.com in any browser and sign in.
  2. DevTools → Application → Cookies → kick.com.
  3. Copy kick_session + XSRF-TOKEN (required), and session_token (needed to claim drops). cf_clearance optional.
  4. Paste below + one or more channels (comma/space-separated).
