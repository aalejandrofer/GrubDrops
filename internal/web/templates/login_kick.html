Kick has no public OAuth. GrubDrops replays your kick.com session directly over a Chrome-fingerprinted HTTP client (utls) — no browser, no sidecar. When discovery logs cloudflare / 401, the cookies have gone stale — re-grab here.

Download the helper

A tiny app that reads your local browser's kick.com cookies and pushes them here — no DevTools, no copy-paste. Grab the build for your OS:

⬇ macOS (Apple Silicon) ⬇ macOS (Intel) ⬇ Windows ⬇ Linux
  1. Make sure you're signed in to kick.com in your everyday browser.
  2. Double-click the downloaded helper. It opens a console and walks you through it.
    macOS / Linux: first run chmod +x the file; macOS may need right-click → Open the first time.
  3. When prompted, paste this Account ID and the Kick channel(s) you want to mine:
account id
{{.AccountID}}
Miner URL defaults to this deployment. You'll also be asked for your admin password.

Manual cookie paste

  1. Open kick.com in any browser and sign in.
  2. DevTools → Application → Cookies → kick.com.
  3. Copy kick_session + XSRF-TOKEN (required), and session_token (needed to claim drops). cf_clearance optional.
  4. Paste below + one or more channels (comma/space-separated).
