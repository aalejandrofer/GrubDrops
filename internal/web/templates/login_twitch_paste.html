Authorize {{.DisplayName}}

Twitch's integrity gate rejects web-issued auth-tokens — gql returns dropCampaigns: null, PubSub LISTEN returns ERR_BADAUTH. Only Android-issued OAuth tokens pass. Three paths below; first one is the only one that fully unlocks drops.

recommended

Device-code login

Mints an Android-issued OAuth token via Twitch's official device-code flow. The miner injects it as the auth-token cookie in the sidecar tab so gql + PubSub both unlock.

Start device-code → Opens activate.twitch.tv with a 6-character code. ~30 seconds.
alt

Helper CLI / pre-built app

Note: cookies snapshotted from your browser are web-issued, so they DO NOT pass integrity (same outcome as the manual paste below). Useful for legacy accounts or debugging only.

one-liner
go run ./cmd/dropsminer-helper twitch {{.AccountID}}
From the repo root. Add --miner URL for a remote deployment.
fallback

Manual cookie paste

  1. Open twitch.tv in any browser and sign in.
  2. DevTools → Application → Cookies → https://www.twitch.tv.
  3. Copy at minimum auth-token.
  4. Paste below. Same integrity caveat as the helper CLI.
